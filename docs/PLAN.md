## Архитектурный обзор

### Два основных режима работы

| Режим | Описание | Датчики | Управление |
|-------|----------|---------|------------|
| **DISTILLATION** (дистилляция) | Простой перегон, два PID-контура | 2 датчика: температура пара в кубе + температура охлаждающей жидкости/конденсата | **Нагрев**: PID → PWM → SSR/реле (electric) или ESP-NOW (gas). **Охлаждение**: PID → PWM → вентилятор на радиаторе **ИЛИ** сервопривод (настраивается из UI) |
| **RECTIFICATION** (ректификация) | Колонна с отбором фракций | 3+ датчиков: column_top, head_selection, body_selection, cooler | **Нагрев**: PID → PWM → SSR/реле (electric) или ESP-NOW с PID (gas). **Охлаждение**: PID → PWM → дефлегматор |

### Подрежимы ректификации

| Подрежим | Описание |
|----------|----------|
| **VAPOR** (отбор по пару) | Регулировка через дефлегматор, управление reflux ratio паром |
| **LIQUID** (отбор жидкой фракции) | Отбор готового продукта жидкостным клапаном/краном |

### Вариативность нагревателя

#### ElectricHeater (электрический ТЭН)
- Контроллер **полностью** управляет нагревом:
  - PID вычисляет выход → PWM (LEDC) → твердотельное реле (SSR) → ТЭН
  - PID охладителя → PWM → **вентилятор на радиаторе** ИЛИ **сервопривод** (тип настраивается из UI)
  - Датчик температуры подключён к **этому же** контроллеру (OneWire шина)
- Пины: `pin_pwm` (нагрев), `pin_cooler` (охладитель)
- Тип охладителя: `COOLER_FAN` (PWM вентилятор) / `COOLER_SERVO` (PWM сервопривод) — настраивается в UI

#### GasHeater (газовый блок, ESP-NOW)
- Контроллер **делегрует** нагрев внешнему блоку:
  - Передача: старт/стоп, мощность 0-100%
  - **Опционально**: PID-коэффициенты шлются на блок (блок сам вычисляет PID)
  - **Ручной режим**: контроллер шлёт фиксированную мощность
  - Датчик температуры газа — **на газовом блоке**, температура приходит в ack
- Связь: ESP-NOW с CRC32, ack, retry до 3 попыток

### Гибкое назначение датчиков

До 4 датчиков DS18B20 на OneWire шине. Каждый датчик настраивается:
- **Адрес** (автоматически определяется при сканировании шины)
- **Роль** (выбирается пользователем): `column_top`, `head_selection`, `body_selection`, `cooler`, `boiler`, `custom`
- **Имя** (человекочитаемое, для UI)
- **Участие в калибровке** (да/нет — галочка в UI)
- **Ручное смещение** (если калибровка отключена)
- **Активен/неактивен** (можно временно исключить из расчётов)

## ⚠️ ТЕХНИЧЕСКИЙ ДОЛГ (требует доработки)

### Критические
- [ ] **Два режима** — `OperationMode: DISTILLATION / RECTIFICATION` не реализованы в `config.h`
- [ ] **Подрежимы ректификации** — `RectSubMode: VAPOR / LIQUID` не реализованы
- [ ] **Тип охладителя** — `CoolerType: FAN / SERVO` не реализован (дистилляция)
- [ ] **Два PID-контура в дистилляции** — нагрев куба + охлаждение конденсата, оба настраиваемые из UI
- [ ] **Гибкое назначение датчиков** — роли захардкожены, нет active/inactive
- [ ] **GasHeater передача PID** — сейчас только power+enable, нет отправки PID-коэффициентов
- [ ] **Ручное управление для GasHeater** — нет режима fixed power
- [ ] **State Machine** — знает только ректификацию, нет режима дистилляции
- [ ] **PID с захардкоженным dt=0.1f** — и в StateMachine, и в ElectricHeater дублируется, dt не измеряется реально

### Средние
- [ ] PID в StateMachine использует захардкоженный dt (баг BUG-008)
- [ ] AI Bridge `set_power` не устанавливает мощность (баг BUG-004)
- [ ] `updatePIDCooler()` нигде не вызывается
- [ ] `getStatusForAI()` возвращает заглушки

## Технические решения
- PSRAM для JSON-документов и логового буфера
