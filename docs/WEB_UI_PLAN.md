# План разработки — Веб-интерфейс SPA

## Задача
Создать полноценный SPA для системы управления ректификационной колонной.

## Файлы для создания
- [ ] `src/ui/web_assets/index.html` — структура с 5 вкладками
- [ ] `src/ui/web_assets/css/style.css` — стили, адаптивность, цветовая схема  
- [ ] `src/ui/web_assets/js/app.js` — логика: WebSocket, графики, команды, UI

---

## Требования

### 1. Без внешних зависимостей
- [x] НЕ использовать CDN (Bootstrap, jQuery и т.д.)
- [x] Всё локально: CSS и JS в отдельных файлах
- [x] Автономная работа (ESP32 раздаёт WiFi, нет интернета)

### 2. WebSocket для real-time обновлений
- [ ] Подключение к `ws://<ip>/ws`
- [ ] Обработка телеметрии
- [ ] Отправка команд

### 3. HTTP API
- [ ] `GET /api/status/snapshot` — начальное состояние
- [ ] `GET /api/config` — настройки
- [ ] `GET /api/log/latest?n=50` — логи
- [ ] `POST /api/command` — команды

---

## Вкладки интерфейса

### 1. Dashboard (Главная)
- [ ] Температуры: 4 датчика с графиками в реальном времени (Canvas)
- [ ] Фаза: визуальный индикатор (цветной бейдж)
- [ ] ΔT: большое число
- [ ] Мощность нагрева: progress bar (0-100%)
- [ ] PID выходы: два индикатора
- [ ] Uptime: таймер
- [ ] Статус подключения: индикатор WebSocket

### 2. Управление
- [ ] Кнопка «Запуск»
- [ ] Кнопка «Аварийная остановка»
- [ ] Кнопка «Сброс аварии»
- [ ] Кнопка «Калибровка»
- [ ] Ручная мощность: ползунок 0-100%
- [ ] Ручная фаза: выпадающий список
- [ ] Статус калибровки

### 3. Настройки
- [ ] Пороги фаз (heads_end, body_end, delta_hysteresis)
- [ ] PID куба (Kp, Ki, Kd, out_min, out_max, integral_max)
- [ ] PID охладителя
- [ ] MQTT настройки
- [ ] WiFi SSID
- [ ] TFT enabled

### 4. Логи
- [ ] Таблица последних записей
- [ ] Колонки: время, фаза, T_top, T_select, T_below, pid_out, heater_type, errors
- [ ] Фильтр по фазе
- [ ] Кнопка «Скачать NDJSON»
- [ ] Автообновление каждые 10 секунд

### 5. AI (опционально)
- [ ] Поле ввода: команда для ИИ-агента
- [ ] Кнопка «Отправить»
- [ ] История команд (последние 20)
- [ ] Статус ИИ-агента
