# Техническое задание: Контроллер дистилляции и ректификации (ESP32-S3)

## Роль
Ты — старший разработчик встраиваемых систем (Embedded C++). Твоя задача — спроектировать и сгенерировать модульный, безопасный код для ESP32-S3 (PlatformIO/Arduino Core) для управления процессом перегонки и ректификации.

## 🔑 Ключевые изменения и логика
1. **Режимы работы (Modes):**
   - **DISTILLATION (Дистилляция):** 
     - Фазы: HEATING → HEADS (отбор голов) → BODY (отбор тела) → FINISH.
     - Управление HEADS: `manual` (кнопка) или `quantitative` (авто по объему).
     - Нет фаз TAILS (хвосты отбираются до конца или по команде).
   - **RECTIFICATION (Ректификация):** 
     - Фазы: HEATING → HEADS → BODY → TAILS → FINISH.
     - Управление переходами (Phase Transition Mode): 
       1. `manual` (вручную).
       2. `delta_temp` (автоматически по дельте Т2-Т3).
       3. `quantitative` (автоматически по объему отбора).
   - **Датчик "Тело" (`body_selection`):** Участвует только в расчете `delta_T` для ректификации. В UI **не отображается** (`display: false`).

2. **ESP-NOW (Разделено на 2 канала):**
   - **Канал 1: GAS CONTROL BLOCK (Управление газом).** 
     - Контроллер отправляет: PID-коэффициенты, уставку темп., старт/стоп.
     - Контроллер получает: Текущую темп. куба, статус блока, текущую мощность газа.
   - **Канал 2: LEVEL & ABV MODULE (Измерение уровня).**
     - Контроллер получает: Уровень в мм (`level_mm`), Крепость (`abv`), Давление (`pressure`), Т-ру жидкости.
     - Логика: Контроллер пересчитывает `level_mm` в `volume_ml` по таблице калибровки.

3. **Логика приема емкости (Collection Tank):**
   - В `settings.json` есть таблица калибровки (mm -> ml).
   - При заполнении (настраиваемый %): Остановка отбора, уведомление в UI.
   - Кнопка "Продолжить" (сброс счетчика текущей тары, продолжение).
   - В логах: Текущий объем, Скорость отбора (мл/мин), Суммарный объем за сессию.

4. **Настройки и Сервисы:**
   - **MQTT:** Полная настройка через UI (Host, Port, Auth, Topic).
   - **OTA:** Возможность обновить прошивку через веб-интерфейс (HTTP Upload).
   - **Журнал (Logging):** NDJSON, ротация по размеру.

Требования к коду
C++17, ESP32-S3, PlatformIO.
Асинхронный веб-сервер (ESPAsyncWebServer).
Жесткое разделение логики ESP-NOW для газа и уровня.
Безопасность (Watchdog, Emergency Stop).

# 💻 Реализация проекта (C++ / PlatformIO)

Ниже приведены ключевые файлы. Код написан так, чтобы быть компилируемым и соответствовать логике.

### 1. `platformio.ini`
Конфигурация среды сборки.

```ini
[env:esp32-s3]
platform = espressif32
board = esp32-s3-devkitc-1
framework = arduino
monitor_speed = 115200

    bblanchon/ArduinoJson @ ^6.21.3
    https://github.com/me-no-dev/ESPAsyncWebServer
    https://github.com/me-no-dev/ESPAsyncTCP
